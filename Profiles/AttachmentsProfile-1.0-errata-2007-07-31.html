er006 error of omission in wsdl/mime schema
Description
The mimeXml element and its corresponding type are absent from the version of the wsdl/mime schema (updated) that is hosted on the WS-I site.
Resolution: 2007-06-05

Restore the mimeXML element and its corresponding type to the following schemas:

  • http://schemas.xmlsoap.org/wsdl/mime/2004-08-24.xsd
  • http://ws-i.org/profiles/basic/1.1/wsdlmime-2004-08-24.xsd

The missing content is:

  <import namespace="http://schemas.xmlsoap.org/wsdl/" schemaLocation="http://schemas.xmlsoap.org/wsdl/" />
  <element name="mimeXml" type="mime:tMimeXml"/>
  <complexType name="tMimeXml"> 
       <complexContent> 
           <extension base="wsdl:tExtensibilityElement"> 
               <sequence /> 
               <attribute name="part" type="NMTOKEN" use="optional" /> 
           </extension> 
       </complexContent> 
   </complexType>
er007 Section 3.12, R2936, is incorrectly stated, and is leading to interoperability problems.
Description
RFC 2046 section 5.5.1 only requires MIME encapsulation boundaries *after the initial boundary* to be preceded with a CRLF. The initial boundary has special status according to the BNF.
Resolution: 2007-03-28

Section 3.12, Change R2936 to: "In a MESSAGE, all MIME encapsulation boundary strings, except the boundary string of the first MIME part, MUST be preceded with the ascii characters CR (13) and LF (10) in that sequence."
